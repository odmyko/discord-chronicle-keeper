# Security Policy

## Supported Versions

Security fixes are provided for the latest commit on `master`.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Report security issues privately to the maintainer with:
- clear description of the issue
- steps to reproduce
- impact assessment
- proposed mitigation (if available)

After receiving a report, the maintainer will:
- acknowledge receipt
- validate and triage severity
- prepare a fix and release
- disclose details after a fix is available

## Scope

This policy covers:
- bot command handling and permission checks
- artifact handling and storage behavior
- dependency and container image security issues

Out of scope:
- vulnerabilities in third-party services (Discord, Docker, LM engines, Whisper models) unless caused by this project integration.
